# ADR-011：多模态图片入库（Phase 4.1）

## 状态

已接受（2026-05-24）

## 背景

Phase 4 出口条件为「图片入库可用」。当前仅支持 PDF/MD/TXT 文本解析；Embedding 端点已支持 vision，但未用于图片文件。

## 决策

1. **上传**：扩展 `DocumentService.upload`，接受 `png/jpg/jpeg/webp/gif/bmp`，`file_type=image`，默认上限 10MB（`MAX_IMAGE_UPLOAD_SIZE`）。
2. **处理管线** `_process_image`：
   - Vision LLM 生成中文描述（`vision_caption_service`，可选 `VISION_CAPTION_MODEL`，默认走 `VOLCENGINE_LLM_MODEL`）。
   - 单 chunk 文本：`[图片文档] {filename}\n\n{caption}`，走现有 `ingest_text_chunks`（门禁/去重/图谱/FTS 与文本一致）。
   - Chroma 向量使用 `EmbeddingService.embed_image(file_path)`，metadata `media_type=image`。
3. **Mock**：`LLM_MOCK_MODE=true` 时 caption 与 embedding 均走确定性 mock，供 CI 与 `verify_phase4_1.py`。
4. **开关**：`MULTIMODAL_IMAGE_ENABLED` 可关闭图片上传。

## 非目标（后续 Phase 4.x）

- PDF 内嵌图 OCR/caption
- SIM-RAG、以图搜图专用 UI
- 会话内临时图片（不入库）

## 验证

```bash
cd backend
python -m pytest tests/test_media_utils.py tests/test_phase4_image_ingest.py -q
python scripts/verify_phase4_1.py
```

## 后果

- 图片块在 FTS/图谱侧按**描述文本**索引，向量侧为**图像 embedding**；跨模态检索依赖 caption 与 image 向量在同一空间（当前火山 multimodal 端点假设成立）。
- 重新启用文档时，image 类型使用 `embed_image` 而非 `embed_query`。
