# ADR-001: 知识缺口队列（Phase 1.1）

**状态**：已采纳  
**日期**：2026-05-26

## 决策

- 表 `knowledge_gaps`，`gap_type` 四类：`RETRIEVAL_MISS` | `USER_PROVIDED` | `USER_CORRECTION` | `KNOWLEDGE_ABSENT`
- `KNOWLEDGE_ABSENT` 默认 `status=manual_required`，禁止 LLM 自动编造入库
- 对话结束后若检索空 / 低分 / 回答含「暂无相关信息」→ `GapService.maybe_enqueue_from_chat`
- API：`GET/POST /api/knowledge-bases/{kb_id}/gaps`，`PATCH .../gaps/{id}/status`
- 前端：`/knowledge-bases/:kbId/gaps` 补全任务页

## 分类规则（启发式 v1）

| 条件 | gap_type |
|------|----------|
| 有纠正文本 | USER_CORRECTION |
| 用户句似事实陈述 | USER_PROVIDED |
| 无检索结果但向量弱命中 | RETRIEVAL_MISS |
| 无检索且向量无弱命中 | KNOWLEDGE_ABSENT |
| 有结果但最高分 < 0.2 | 同上弱命中逻辑 |
